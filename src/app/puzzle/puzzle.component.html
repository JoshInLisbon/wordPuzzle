I was not able to build the word search puzzle using Angular. However, I would like to demonstrate that, though I am not familiar with Angular, I am a competent JavaScript developer (who will pick it up rapidly with just a little bit of coaching). 

To do this I have written a set of JavaScript functions which returns an array objects to the console containing the grid number and letter for a word search puzzle, laying out words all directions: horizontal, up, down, upDiagonal, downDiagonal. 

The solution is scalable, handling a grid of any n*n size and any number of words (within reason); refreshable, changing after every refresh; able to handle complexity, allowing words to overlap if they share the same letter; and efficient, with words only 'spawning' (and subsequently checking for conflicts with established words) in squares where they have the room to exist. To play with the solution edit lines 15 and 16 in 'puzzle.component.ts'.

15. let n = 8;
16. let words = ["stop", "healthy", "happy", "save", "breath"];
